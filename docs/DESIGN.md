# UI/UX 设计规范文档 (DESIGN.md)

## 1. 设计概述 (Design Overview)

本设计文档旨在为局域网互动系统提供统一的视觉风格和交互规范。设计核心原则：
*   **大屏端 (Public Screen)**: 沉浸式、PPT演示风格、远距离可读性强、动效流畅。
*   **移动端 (Mobile)**: 极简操作、响应迅速、防止误触、明确的状态反馈。
*   **管理端 (Admin/Host)**: 功能清晰、控制便捷、数据可视化。

## 2. 视觉规范 (Visual Identity)

### 2.1 配色方案 (Color Palette)
*   **主色调 (Primary)**: `#0052D9` (科技蓝) - 用于按钮、高亮状态。
*   **大屏背景 (Large Screen BG)**: `#181818` (深灰/黑) - 适合投影，减少刺眼光线，提升对比度。
*   **移动端背景 (Mobile BG)**: `#F5F5F5` (浅灰) - 清晰清爽。
*   **辅助色**:
    *   成功/正确: `#00A870` (绿)
    *   错误/警告: `#E34D59` (红)
    *   倒计时/强调: `#ED7B2F` (橙)

### 2.2 字体排版 (Typography)
*   **大屏**: 
    *   标题: 64px+, 加粗, 无衬线字体 (PingFang SC, Microsoft YaHei UI).
    *   正文: 32px+, 确保后排观众可见。
*   **移动端**:
    *   标准 Web 字体大小 (14px - 18px)，输入框文字需足够大 (16px+) 以防缩放。

## 3. 界面设计 (Interface Design)

### 3.1 大屏端 (Large Screen) - 1920x1080

#### A. 待机/签到页 (Standby/Check-in)
*   **布局**: 左右分栏 或 中心聚焦。
*   **左侧/中心**: 巨大的动态二维码 (包含 Session 信息)，下方附文字“扫描二维码加入”。
*   **右侧/背景**: 
    *   动态漂浮的已签到用户头像/昵称 (Avatar Wall)。
    *   右上角: 实时在线人数 (Counter)。
    *   底部: WiFi 连接指引 (SSID & Password)。
*   **动效**: 新用户签到时，头像从二维码处飞入头像墙。

#### B. 互动容器页 (Interaction Container)
*   **顶部栏 (Header)**: 高度 80px，显示当前环节标题 (如“找数字规律”)、倒计时 (巨大的数字)。
*   **内容区 (Main)**: 占据屏幕 80% 区域，用于加载插件内容。
*   **底部栏 (Footer)**: 进度条或当前参与人数统计 (如 "150/200 已提交")。

#### C. 具体互动示例 UI
*   **找数字规律 (Number Pattern)**:
    *   **背景**: 深色星空或纯色背景。
    *   **元素**: 数字以高亮气泡或悬浮文字形式展示。
    *   **阶段1 (散点)**: 数字随机分布，呼吸灯效果。
    *   **阶段2 (象限)**: 屏幕被十字线划分为四个区域，数字按象限顺序依次亮起/出现。
    *   **阶段3 (行序)**: 类似代码流或矩阵，整齐排列。
    *   **结果页**: 柱状图显示正确率，Top 5 最快答对者名单 (金银铜牌动效)。

### 3.2 移动端 (Mobile) - 响应式 H5

#### A. 登录/注册页 (Login)
*   **布局**: 垂直居中。
*   **元素**: 公司 Logo，输入框 (姓名)，大尺寸“加入”按钮。
*   **反馈**: 提交后显示 Loading，成功后跳转等待页。

#### B. 等待/导航页 (Lobby)
*   **顶部**: 个人信息卡片 (昵称 + 随机ID)。
*   **中部**: 
    *   当前状态提示 (如“等待主持人开启下一环节...”)。
    *   导航 Tab: [互动] [议程] [我的]。
*   **议程 Tab**: 时间轴样式展示会议流程。

#### C. 互动输入页 (Interaction Input)
*   **设计原则**: 减少干扰，聚焦操作。
*   **找数字规律插件**:
    *   **顶部**: 题目提示 (如 "请找出缺失的数字")。
    *   **中部**: 
        *   标准输入框 (type="tel" 或 "number")，自动弹出数字键盘。
        *   JS 校验：实时屏蔽非数字字符输入。
        *   多个输入框 ( _ _ _ ) 自动跳转 (可选优化)。
    *   **底部**: “提交”按钮 (提交后变为“等待结果”状态)。
    *   **反馈**: 提交正确/错误在本地不立即显示，待大屏统一揭晓，或仅显示“已提交”。

### 3.3 管理/主持人端 (Host Control) - Mobile/Tablet

#### A. 控制面板 (Control Panel)
*   **布局**: 卡片式流布局。
*   **功能区**:
    *   **流程控制**: [上一环节] [开始互动] [结束互动] [下一环节] (大按钮，防止误触)。
    *   **状态监视**: 实时显示当前环节的提交人数、正确率。
    *   **插件选择**: 下拉菜单或列表选择临时插入的互动内容。

## 4. 交互流程 (User Flow)

### 4.1 找数字规律 (Find Number Pattern) 流程
1.  **准备**: 主持人点击“加载找数字游戏”。大屏显示游戏规则封面。手机端显示“游戏即将开始”。
2.  **倒计时**: 大屏显示 3, 2, 1。
3.  **进行中 (阶段1)**: 
    *   大屏: 显示数字云 (30秒)。
    *   手机: 震动提示，输入框解锁，允许输入。
4.  **结束/统计**: 
    *   30秒到 或 主持人点击“结束”。
    *   大屏: 锁定画面，数字云消失，出现统计图表 (正确率) 和 答案揭晓 (缺失的数字高亮显示)。
    *   手机: 显示“本轮结束”，弹出用户本轮得分。
5.  **下一轮**: 主持人点击“下一轮 (象限模式)”，重复上述步骤。

## 5. 插件化UI适配规范 (Plugin UI Spec)
为了保证扩展性，所有互动插件的前端必须遵循以下 DOM 结构与 CSS 命名空间：
*   **大屏插件**: 需适配 `1920x1080`，使用 `rem` 或 `vw/vh` 单位。容器 ID `#plugin-container-large`。
*   **手机插件**: 需适配 `375px` 宽度的流式布局。容器 ID `#plugin-container-mobile`。
*   **资源加载**: 必须通过相对路径 `./assets/` 引用图片。
